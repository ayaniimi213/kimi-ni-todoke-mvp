# 君に届け (Kimi ni Todoke) - MVP Prototype

**「荷物を待つのではなく、荷物があなたに歩み寄る」**

配送先を固定された住所ではなく、ユーザーの予定や現在地に合わせて柔軟に決定する配送サービスのUXプロトタイプです。

## 🚀 サービス概要
現代の忙しいユーザーにとって、自宅で荷物を待つ時間は制約となります。
このサービスは、ユーザーの「現在地」「次の予定」「移動ルート」を解析し、最もストレスなく立ち寄れる受取場所を提案します。

**本リポジトリは、MVP（最小機能製品）として「最適な受取場所を提案・選択する」体験に特化して実装されています。**

## ✨ このプロトタイプで検証したいこと
このMVPでは、実際の配送・決済・管理機能はあえて実装せず、以下のコア体験の検証に集中しています。

- **動線上の提案**: 単なる「近い順」ではなく、移動ルートの邪魔にならない（遠回りが少ない）場所への提案に価値があるか。
- **提案の納得感**: 「なぜそこがおすすめなのか」という理由の提示が、ユーザーの意思決定を助けるか。
- **柔軟な受取体験**: 予定の合間にサクッと受け取るというスタイルが、ユーザーの利便性を向上させるか。

---

## 🛠 機能紹介
1. **ユーザー情報の入力**
   - 現在地（GPSまたは選択肢）
   - 次の予定地と予定時刻
   - 受取希望時間
2. **おすすめ受取場所の計算**
   - ルールベースの独自スコアリング（Haversine式を用いた距離計算）
   - 現在地→受取場所→予定地の「遠回り度」を評価
3. **提案結果の表示**
   - 上位3件をランキング形式で表示
   - アルゴリズムに基づいた「おすすめの理由」を自然言語で表示
4. **受取場所の決定**
   - 提案の中から1つを選択し、受取フローをシミュレーション

---

## 🧮 スコアリングのロジック
本プロトタイプでは、以下の要素を組み合わせてスコアを算出しています。
- **移動コスト**: 現在地からスポット、スポットから予定地までの総徒歩距離。
- **遠回り度 (Detour Factor)**: 直線ルートからどれだけ外れているかを計算。
- **時間適合性**: スポットの営業時間と、ユーザーの受取希望時間の合致。
- **近接ボーナス**: 次の予定地のすぐ近く（300m以内）にあるスポットを優遇。
